│ 1. CREDIBLE NEUTRALITY │
│ "A structure is credibly neutral if, just by looking │
│ at its design, it is easy to see that it does not │
│ discriminate for or against anyone." │
│ │
│ → Policies do adr-ledger são públicas (Radicle) │
│ → Decisões são verificáveis (Algorand) │
│ → Enforcement é determinístico (OPA + NixOS) │
│ → Nenhum agente tem backdoor │
│ │
│ 2. d/acc (defensive/decentralized acceleration) │
│ "Build technology that is structurally more favorable │
│ to defense than offense." │
│ │
│ → Fail-closed por design │
│ → Agentes são sandboxed no OS │
│ → Rate limiting em 3 camadas (OPA, Chain, systemd) │
│ → Circuit breaker on-chain │
│ │
│ 3. SOULBOUND IDENTITY │
│ Identidade de agentes não-transferível, │
│ vinculada a histórico de ações │
│ │
│ → Agente acumula reputação verificável │
│ → Histórico é imutável (Algorand) │
│ → Identidade cripto (Radicle DID + on-chain) │
│ → Não pode "comprar" a reputação de outro agente │
│ │
│ 4. PLURALITY / MECANISMOS QUADRÁTICOS │
│ Decisões de governança não são 1-token-1-vote │
│ │
│ → Approval weight por contexto (ABAC) │
│ → Quadratic voting em policy changes │
│ → Conviction voting para mudanças graduais │
│ │
