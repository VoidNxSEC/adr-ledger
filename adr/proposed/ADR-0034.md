## Context

After a comprehensive validation session on the Phantom v2.0 project, test coverage was raised from **70.00% (450 tests)** to **73.80% (496 tests)** with 0 failures. However, several modules remain below 60% coverage due to structural dependencies that prevent straightforward unit testing.

This ADR serves as a **living registry** of coverage gaps, their root causes, and the recommended approach for each — providing project-wide visibility for future development decisions.

## Decision

Maintain this registry as the authoritative source for coverage planning. Priority order for next sessions:

1. **`rag/cortex_embeddings.py` (53%)** — Mock `SentenceTransformer` and `FAISSVectorStore` init; test `EmbeddingGenerator`, `EmbeddingManager` business logic
2. **`analysis/ai_analyzer.py` (54%)** — Mock `LlamaCppProvider.generate()`; test config parsing, result models, prompt building
3. **`api/judge_api.py` (60%)** — Mock `PhantomSentinel` and `OracleExplainer`; test `JudgmentEngine` validation and scoring
4. **`pipeline/phantom_dag.py` (63%)** — Test `PhantomPipeline` with in-memory DAG; mock file system operations
5. **`cli/main.py` (61%)** — BLOCKED until CLI commands are implemented (all stubs)

## Embedding Server Status (ADR-0028)

The `cerebro-embeddings` systemd service (`~/.config/systemd/user/cerebro-embeddings.service`) is:
- **Imperative**: not managed by Nix/home-manager
- **Disabled**: `systemctl --user is-enabled` → disabled (does NOT survive reboot)
- **Stopped**: port 8001 not listening, no process running
- **Migration pending**: ADR-0028 proposes home-manager declarative migration

**No immediate action required** — the service won't restart on reboot. ADR-0028 implementation should be scheduled when CEREBRO embedding functionality is needed again.
