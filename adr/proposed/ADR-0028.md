## Context

The `cerebro-embeddings` service (Vertex AI `text-embedding-004`, 768 dims, port 8001) is currently defined as an **imperative systemd user service** at `~/.config/systemd/user/cerebro-embeddings.service`. This file is invisible to NixOS/home-manager and breaks the declarative contract.

**Evidence of the problem:** The service crash-looped 1630+ times because the `WorkingDirectory` path (`/home/kernelcore/arch/cerebro`) was stale after the repository moved to `/home/kernelcore/master/cerebro`. A declarative Nix definition would have caught this at build time.

The service runs: `nix develop --command python -m uvicorn phantom.core.embedding_server:app --host 0.0.0.0 --port 8001`

It depends on the cerebro flake's devShell (fastapi, uvicorn, vertexai, torch-bin, google-auth) and GCP credentials loaded via SOPS at `/run/user/1001/secrets/gcp-credentials.env`.

## Decision

Migrate `cerebro-embeddings.service` to a declarative home-manager `systemd.user.services` definition at `/etc/nixos/hosts/kernelcore/home/cerebro-embeddings.nix`, imported by `home.nix`. Remove the imperative service file after verification.
