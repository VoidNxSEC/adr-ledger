## Context

O Spectre Fleet está production-ready. Phase 0→4 completas, incluindo:
- `spectre-core`, `spectre-events`, `spectre-proxy`, `spectre-secrets`, `spectre-observability` — 5 crates estáveis
- NATS JetStream com reconexão automática, circuit breaker, RBAC, Argon2id KDF
- Chaos Engineering validado (Phase 4): NATS down/up, graceful shutdown, cascading failures
- Nix module nativo (`nix/services/nats/`) pronto para importação por outros projetos
- Helm chart (17 arquivos) + Nix-native Kubernetes orchestration

O **owasaka** foi o primeiro domain service a integrar via NATS como parte do ADR-0050 — publicando `network.asset.discovered.v1` e `network.dns.query.v1`. O `ai-agent-os` também publica `system.metrics.v1`.

Os demais domain services do umbrella (`securellm-bridge`, `ml-offload-api`, `cerebro`) ainda comunicam via HTTP direto, sem passar pelo barramento. O `spectre-observability` existe como biblioteca de inicialização mas o **stream processor** (wildcard NATS subscriber para coleta de métricas cross-service) não foi implementado.

O Spectre não foi tocado ativamente desde março/2026 — está estável mas parado na fronteira entre "infraestrutura pronta" e "plataforma de observabilidade operacional".

## Decision

Registrar como ADR proposto/pendente o trabalho de integração dos domain services e implementação do pipeline de observabilidade do `spectre-observability`. Este ADR permanece **aberto** enquanto o trabalho não for iniciado.

A ordem de implementação recomendada:

```
S1 (securellm-bridge NATS) → S2 (ml-offload-api NATS) → S3 (cerebro NATS)
→ S4 (stream processor wildcard) → S5 (TimescaleDB schema)
→ S6 (Neo4j dependency graph) → S7 (ML baseline + anomaly detection)
→ S8 (Grafana dashboards) → S9 (NixOS module) → S10 (proxy Phase 2)
```

A integração de cada service é independente — pode ser feita em qualquer ordem. A dependência real é que S4→S7 requerem pelo menos 2 services publicando eventos para ter dados significativos.

## Rationale

### Estado atual do Spectre (2026-03-26)
- **Infrastructure**: ✅ production-ready
- **Domain service integration**: ⚠️ apenas owasaka + ai-agent-os integrados
- **Observability pipeline**: ❌ stream processor não implementado
- **ML anomaly detection**: ❌ não iniciado
- **NixOS native deployment**: ⚠️ Nix module existe mas docker-compose ainda é o modo primário

### Por que manter aberto
O Spectre está estável e não bloqueia outros projetos. A integração dos domain services é trabalho incremental que pode ser atacado quando houver bandwidth — não é urgente, mas é o passo que transforma o Spectre de "event bus disponível" para "plataforma de observabilidade operacional".

## Consequences

### Positive
- Visibilidade unificada de todos os serviços em Grafana
- Rastreabilidade de custos LLM por serviço/request
- Detecção automática de anomalias (latência, taxa de erro, VRAM) via ML
- Dependency graph em Neo4j para debugging de propagação de falhas

### Negative
- Cada integração requer refactor no service correspondente
- spectre-observability ML requer período de coleta de baseline

## Implementation

### S1 — securellm-bridge: NATS integration
**Files**: `securellm-bridge/crates/core/src/` + `Cargo.toml`
- Adicionar `spectre-events` como dependência
- Publicar `llm.request.v1` ao início de cada request
- Publicar `llm.response.v1` com latência e token count
- Publicar `cost.incurred.v1` com custo estimado por provider

### S2 — ml-offload-api: NATS integration
**Files**: `ml-offload-api/`
- Publicar `inference.request.v1` e `inference.response.v1`
- Publicar `vram.status.v1` a cada mudança significativa de VRAM

### S3 — cerebro: NATS integration
**Files**: `cerebro/`
- Publicar `cognition.query.received.v1` e `cognition.insight.generated.v1`
- Publicar `document.indexed.v1` ao indexar novos documentos

### S4 — spectre-observability: stream processor
**File**: `spectre/crates/spectre-observability/src/stream.rs` (novo)
- Wildcard NATS subscriber: `*.*.*.*`
- Persiste eventos no TimescaleDB
- Atualiza Neo4j dependency graph

### S7 — ML anomaly detection
**File**: `spectre/crates/spectre-observability/src/ml.rs` (novo)
- Baseline: média + desvio padrão por subject/service nos últimos 7 dias
- Alert: z-score > 3σ em latência ou taxa de erro
- Fallback: regras threshold enquanto baseline não está disponível
